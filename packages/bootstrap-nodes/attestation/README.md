# Attestation

Wattz supports three TEE evidence formats plus a software-only fallback.
This directory contains the shell helpers used by bootstrap nodes to
generate the evidence before it is forwarded to the gateway.

Verification lives in the `wattz-compute-verifier` Rust crate; the
scripts here are only responsible for producing the evidence in the wire
format the verifier expects.

## Intel SGX DCAP

`generate-sgx-quote.sh` invokes `gramine-sgx-attest` (or `sgx_qgs_client`
on non-gramine deployments) with the ed25519 public key of the node as
the report data. The resulting binary quote is base64-encoded and
written to `$WATTZ_NODE_ATT_QUOTE_FILE`.

Prerequisites:

* SGX-enabled CPU (Coffee Lake or newer with FLC support).
* `sgx-dcap-driver` kernel module loaded.
* Intel PCS collateral fetched (see the Intel docs).

## NVIDIA Confidential Computing

`generate-nvidia-cc.sh` invokes `nvattest` (bundled with the NVIDIA
Confidential Computing SDK) with a caller-supplied 32-byte nonce. It
returns the raw evidence bundle that the compute verifier's
`verify_nvidia_cc_quote` accepts.

Prerequisites:

* NVIDIA Hopper (H100 / H200) or Blackwell (B200 / GB200) GPU.
* Driver with confidential computing support (r550+).
* `nvattest` binary in `PATH`.

## AMD SEV-SNP

SEV-SNP evidence is generated by `snpguest` inside the guest. Wattz does
not ship an AMD-specific helper because `snpguest report --data <hex>`
is already the canonical entry point.

## Software-only nodes

Nodes without TEE hardware run without attestation. The node runtime
still signs every response with an ed25519 key so the receipt is bound
to the specific node, but the gateway records `attestation_type =
"software"` and the routing engine downgrades the node's reputation
weight. Consumers who require TEE evidence can filter with
`?require_tee=true` on the routing request.
